# Markdown Editor

Mac 风格的 Markdown 编辑器，支持实时预览、可视化插入、代码高亮、数学公式、一键复制富文本。

**在线使用：[markdowneditor.cloud](https://markdowneditor.cloud/)**

---

## 使用方法

### 界面布局

顶部标题栏提供所有主要操作，编辑区与预览区可自由切换视图，工具栏包含常用格式和插入按钮。

### 顶部按钮

| 按钮 | 说明 |
|------|------|
| Edit / Split / Preview | 切换纯编辑、分栏、纯预览三种视图 |
| Open | 打开本地 `.md` / `.txt` 文件 |
| Save | 保存为 `.md` 文件，快捷键 `Ctrl+S` |
| Export | 导出为独立 `.html` 文件 |
| Copy ▾ | 下拉选择：**微信公众号**富文本 / **52pojie BBCode** |
| Guide | 使用指南，含 Markdown 语法速查和功能说明 |
| Reset | 清空所有本地数据，恢复初始状态 |
| ⚙️ | 图片存储方式 & 界面语言设置 |
| ⊞ / ⊡ | 专注模式（隐藏工具栏） |
| 🌙 / ☀️ | 切换暗色 / 亮色主题 |

分栏模式下，中间分割线可左右拖拽调整编辑区与预览区的比例。

### 工具栏

- **格式**：H1 / H2 / H3、加粗、斜体、删除线、行内代码
- **块元素**：引用、代码块、分割线、无序/有序/任务列表
- **插入**：链接、图片 URL
- **⊞ 表格**：可视化选择行列数，填写内容后自动生成 Markdown 表格
- **∑ 公式**：数学公式库，含希腊字母、运算符、常用模板，支持行内和块级插入
- **✦ 可视化插入**：Word 风格所见即所得编辑器，详见下方

### 可视化插入

不需要记 Markdown 语法，像 Word 一样写内容，自动转换为标准 Markdown 格式。

1. 点击工具栏 **✦ 可视化插入**
2. 在编辑区输入内容
3. **选中文字** → 点击格式按钮（加粗、标题、链接等）应用样式
4. 点击右下角 **确认插入 ✓**，内容转为 Markdown 插入到编辑器光标处

支持：H1–H3、加粗、斜体、删除线、行内代码、引用、无序/有序列表、链接、表格、图片（URL 或本地上传）。

**粘贴截图**：在可视化编辑器内 Ctrl+V 粘贴截图，自动处理并插入。

**草稿自动保存**：关闭弹窗后重新打开自动恢复上次内容；点击「确认插入」后草稿清除。

### 图片粘贴

直接在编辑器中 Ctrl+V 粘贴截图，自动转换并插入 Markdown 图片语法。Base64 长串在编辑器中自动折叠为一行，右键可展开/收起。

⚙️ 设置中可切换三种存储方式：

| 方式 | 说明 |
|------|------|
| PokerS 链接（默认） | 图片上传后生成短链接，在线可访问 |
| 内嵌 Base64 | 图片嵌入文件，离线可用，文件随图片增大 |
| 本地文件引用 | 图片以相对路径引用，需与 .md 文件同目录 |

### 复制到微信公众号

点击 **Copy ▾ → 微信公众号**，复制带样式的富文本，直接粘贴到微信公众号编辑器，保留标题、代码块、表格、数学公式等样式。

### 复制到 52pojie

点击 **Copy ▾ → 52pojie BBCode**，复制 BBCode 格式，在吾爱破解论坛编辑器切换到「源码」模式后粘贴。支持代码高亮（`[mw_shl_code]`）和表格。

### 快捷键

| 功能 | 快捷键 |
|------|--------|
| 保存 | `Ctrl+S` / `Cmd+S` |
| 加粗 | `Ctrl+B` |
| 斜体 | `Ctrl+I` |

### 自动保存

内容每隔 1 秒自动保存到浏览器本地存储，刷新后自动恢复。状态栏左侧显示最后保存时间。

## 本地部署

### 环境准备（仅首次）

1. 下载安装 [Node.js LTS](https://nodejs.org)
2. 验证安装：
   ```bash
   node -v
   npm -v
   ```
3. 安装项目依赖：
   ```bash
   npm install
   ```
   > 速度慢可先切换镜像：`npm config set registry https://registry.npmmirror.com`

### 启动

```bash
npm run dev
```

浏览器打开 `http://localhost:5173` 即可。关闭时在终端按 `Ctrl+C`。

### 构建

```bash
npm run build
```

产物在 `dist/` 目录，可部署到任意静态托管服务。

### 图片上传服务（可选）

项目包含 `cloudflare-worker/` 子目录，是可独立部署的 Cloudflare Worker + R2 图片托管服务。详见 `cloudflare-worker/README.md`。

## 常见问题

**端口 5173 被占用？** Vite 会自动切换到下一个可用端口，按终端提示的地址打开即可。

**关掉终端还能用吗？** 不能，必须保持终端运行。建议最小化而不是关闭。
